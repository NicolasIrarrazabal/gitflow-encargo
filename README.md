Se usa GitFlow porque ayuda a organizar mejor el trabajo en equipo, ya que se trabaja con distintas ramas para cada cosa.
Por ejemplo, hay ramas para desarrollar nuevas funcionalidades (feature), para arreglar errores urgentes (hotfix) y una rama principal de integracion (develop).
Esto permite tener un mejor control de versiones y mantener más estable el codigo que está en produccion.
